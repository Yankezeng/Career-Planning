# 大学生职业规划平台 - 项目详细技术方案

## 一、项目总体架构

### 1.1 系统分层架构

系统采用经典的分层架构设计，确保各层职责清晰、低耦合、高内聚：

```
┌─────────────────────────────────────────────────────────┐
│                    表现层 (Presentation Layer)            │
│  Vue 3 SPA  │  移动端适配  │  ECharts可视化  │  响应式设计  │
├─────────────────────────────────────────────────────────┤
│                    网关层 (Gateway Layer)                 │
│         Nginx反向代理  │  负载均衡  │  静态资源服务         │
├─────────────────────────────────────────────────────────┤
│                    应用层 (Application Layer)             │
│   RESTful API  │  认证授权  │  业务编排  │  异常处理        │
├─────────────────────────────────────────────────────────┤
│                    服务层 (Service Layer)                 │
│  用户服务  │  学生服务  │  岗位服务  │  匹配服务  │  AI服务   │
├─────────────────────────────────────────────────────────┤
│                  AI Agent层 (Agent Layer)                 │
│  Chat Agent  │  Match Agent  │  Profile Agent  │  ...     │
├─────────────────────────────────────────────────────────┤
│                    领域层 (Domain Layer)                  │
│   用户领域模型  │  学生领域模型  │  岗位领域模型  │  匹配领域模型 │
├─────────────────────────────────────────────────────────┤
│                  基础设施层 (Infrastructure Layer)        │
│  PostgreSQL  │  Redis  │  向量数据库  │  文件系统  │  LLM API │
└─────────────────────────────────────────────────────────┘
```

### 1.3 核心设计原则

1. **模块化设计**：各功能模块低耦合，可独立测试和部署，遵循单一职责原则
2. **依赖注入**：通过构造函数和FastAPI的Depends注入依赖，便于单元测试和Mock
3. **异常统一处理**：全局异常处理器，统一错误响应格式，提供详细的错误码和描述
4. **配置外部化**：所有配置项通过环境变量管理，支持多环境切换(dev/test/prod)
5. **降级容错**：AI服务不可用时自动降级到Mock模式，保证核心功能可用性99.9%

## 二、核心技术路径

### 2.1 多Agent协作架构

#### 2.1.1 Agent编排引擎

系统采用**Supervisor-Worker**模式的Agent编排架构：

```
                    ┌─────────────────┐
                    │ Supervisor Agent│
                    │   (任务调度器)   │
                    └────────┬────────┘
                             │
            ┌────────────────┼────────────────┐
            │                │                │
     ┌──────┴──────┐  ┌──────┴──────┐  ┌──────┴──────┐
     │ Chat Agent  │  │ Match Agent │  │Profile Agent│
     └─────────────┘  └─────────────┘  └─────────────┘
```

**编排流程**：
1. 接收用户请求，Supervisor进行意图分析
2. 根据意图类型和复杂度，拆分任务子图
3. 分发子任务到对应的Worker Agent
4. 收集各Agent执行结果，进行结果聚合
5. 生成最终响应返回给用户

#### 2.1.2 Agent工具注册中心

采用**工具注册表**模式，支持动态工具注册和发现：

```python
# 工具注册中心示例
class ToolRegistry:
    _tools: Dict[str, BaseTool] = {}
    
    @classmethod
    def register(cls, tool: BaseTool):
        cls._tools[tool.name] = tool
    
    @classmethod
    def get_tool(cls, name: str) -> Optional[BaseTool]:
        return cls._tools.get(name)
    
    @classmethod
    def list_tools(cls) -> List[str]:
        return list(cls._tools.keys())
```

### 2.5 人岗匹配算法

#### 2.5.1 多维度评分模型

采用**六维能力模型**进行综合评分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 技能匹配度 | 30% | 专业技能与岗位要求的匹配程度 |
| 教育背景 | 15% | 学历、专业、学校与岗位的匹配度 |
| 经验匹配度 | 20% | 工作/项目经验与岗位要求的匹配度 |
| 软技能 | 15% | 沟通能力、团队协作等软实力 |
| 发展潜力 | 10% | 学习能力、成长空间评估 |
| 兴趣匹配 | 10% | 个人职业兴趣与岗位方向的契合度 |

**计算公式**：
```
总分 = Σ(维度得分 × 维度权重)
```

### 4.2 状态管理（Pinia）

```javascript
// stores/student.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { studentApi } from '@/api/student'

export const useStudentStore = defineStore('student', () => {
  // 状态
  const profile = ref(null)
  const skills = ref([])
  const loading = ref(false)
  
  // 计算属性
  const completeness = computed(() => {
    if (!profile.value) return 0
    const fields = ['realName', 'university', 'major', 'grade']
    const filled = fields.filter(f => profile.value[f]).length
    return (filled / fields.length) * 100
  })
  
  // 异步方法
  async function fetchProfile() {
    loading.value = true
    try {
      profile.value = await studentApi.getProfile()
      skills.value = profile.value.skills || []
    } finally {
      loading.value = false
    }
  }
  
  async function updateProfile(data) {
    profile.value = await studentApi.updateProfile(data)
  }
  
  return { profile, skills, loading, completeness, fetchProfile, updateProfile }
})
```

```javascript
import { createRouter, createWebHistory } from 'vue-router'
import { useAuthStore } from '@/stores/auth'

const routes = [

    children: [

        path: 'dashboard',

        meta: { requiresAuth: true }
      },
      {
        path: 'profile',
        component: () => import('@/pages/profile/index.vue'),
        meta: { requiresAuth: true }
      }
    ]
  },
  {
    path: '/login',
    component: () => import('@/pages/auth/login.vue')
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

// 全局前置守卫
router.beforeEach(async (to, from, next) => {
  const authStore = useAuthStore()
  
  if (to.meta.requiresAuth && !authStore.isAuthenticated) {
    next({ path: '/login', query: { redirect: to.fullPath } })
  } else if (to.path === '/login' && authStore.isAuthenticated) {
    next('/dashboard')
  } else {
    next()
  }
})

export default router
```

#### 6.3.1 向量检索服务

```python
class VectorSearchService:
    def __init__(self, embedding_model, vector_db):
        self.embedding_model = embedding_model
        self.vector_db = vector_db
    
    async def search(self, query: str, top_k: int = 5) -> List[Document]:
        # 1. 生成查询向量
        query_embedding = await self.embedding_model.encode(query)
        
        # 2. 向量相似度检索
        results = await self.vector_db.search(query_embedding, top_k=top_k * 2)
        
        # 3. Cross Encoder重排序
        reranked = await self.rerank(query, results, top_k)
        
        return reranked
    
    async def rerank(self, query: str, docs: List[Document], top_k: int) -> List[Document]:
        pairs = [(query, doc.content) for doc in docs]
        scores = self.cross_encoder.predict(pairs)
        
        # 按分数排序
        ranked = sorted(zip(docs, scores), key=lambda x: x[1], reverse=True)
        return [doc for doc, _ in ranked[:top_k]]
```

## 十一、关键技术难点与解决方案

### 11.1 简历解析准确率

**问题**：不同格式的简历结构差异大，规则提取准确率低

**解决方案**：

1. **多引擎融合**：规则提取 + LLM补全
   - 使用正则表达式提取结构化字段（手机号、邮箱、URL等）
   - 使用NER模型识别实体（人名、学校、公司等）
   - 使用LLM补全规则提取遗漏的字段

2. **OCR优化**：针对简历特点训练OCR模型
   - 使用PaddleOCR进行中文OCR识别
   - 针对简历常见布局进行预处理（去噪、纠偏）

3. **后处理校验**：手机号、邮箱等字段格式校验
   ```python
   def validate_resume_data(data: ResumeData) -> List[str]:

           errors.append("邮箱格式不正确")
   ```

4. **人工反馈**：用户修正结果用于模型迭代
   - 提供简历信息编辑界面
   - 收集用户修正数据
   - 定期更新提取模型

### 11.2 人岗匹配精度

**问题**：简单的关键词匹配无法准确评估能力匹配度

**解决方案**：

1. **多维度评分**：六维能力模型，加权计算

## 十三、总结

本项目采用现代化的技术栈，构建了一个功能完善的大学生职业规划平台。系统具有以下特点：

### 技术亮点

1. **多Agent协作架构**：15个专业Agent各司其职，Supervisor统一调度，实现复杂的职业规划场景
2. **智能匹配算法**：六维能力模型+语义相似度计算，提供精准的人岗匹配
3. **RAG增强生成**：向量检索+Cross Encoder重排序+LLM生成，提供高质量的对话体验
4. **降级容错设计**：AI服务不可用时自动降级，保证核心功能可用

### 业务价值

1. **学生端**：提供职业规划指导、岗位匹配、简历优化、面试辅导等一站式服务
2. **企业端**：提供人才筛选、招聘辅助、人才画像等企业服务
3. **管理端**：提供数据分析、系统监控、内容治理等管理能力

### 未来展望

1. **AI能力增强**：引入更多专业模型，提升匹配精度和对话质量
2. **移动端适配**：开发移动端App或小程序
3. **社交功能**：增加学长学姐交流、导师咨询等社交功能
4. **数据洞察**：基于大数据分析提供行业趋势、薪资预测等深度洞察
